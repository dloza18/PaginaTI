Puertos de red

Los puertos en TCP (Transmission Control Protocol) y UDP (User Datagram Protocol) son puntos finales en un sistema informático que permiten la comunicación entre diferentes aplicaciones en una red.

Puertos TCP

TCP es un protocolo de comunicación orientado a la conexión que garantiza la entrega y el orden de los datos. Los puertos TCP van desde 0 a 65535, donde los puertos del rango 0-1023 están reservados para servicios bien conocidos y estándar.

Puertos UDP

UDP es un protocolo de comunicación no orientado a la conexión que no garantiza la entrega de datos. Los puertos UDP también van de 0 a 65535.

  • Puerto 53: DNS (Domain Name System)
  • Puerto 69: TFTP (Trivial File Transfer Protocol)

Protocolo HTTP

El protocolo HTTP (Hypertext Transfer Protocol) utiliza el puerto 80 para la comunicación por defecto en la capa de transporte TCP/IP. Cuando un navegador web envía una solicitud HTTP a un servidor web sin especificar un puerto diferente, se asume que se está utilizando el puerto 80.

Es importante tener en cuenta que, si se desea utilizar una conexión segura a través de HTTPS (HTTP Secure), se utiliza el puerto 443 por defecto. HTTPS es la versión segura de HTTP que cifra la comunicación entre el cliente y el servidor, proporcionando un nivel adicional de seguridad.

    En Resumen:

  • HTTP (no seguro): Puerto 80
  • HTTPS (seguro): Puerto 443

Protocolo HTTPS

Protocolo de transferencia de hipertexto seguro (HTTPS) es una versión segura del protocolo HTTP que utiliza el SSL /TLS protocolo para cifrado y autenticación.

HTTPS utiliza una combinación de dos protocolos de comunicación (HTTP+SSL/TLS) que hace que cualquier tipo de información que se transmita en la red sea cifrada y nadie pueda acceder a ella, únicamente navegador y servidor web.

    En Resumen:

  • HTTPS (seguro): Puerto 443
  • HTTP (no seguro): Puerto 80

Protocolo SMTP

El protocolo SMTP (Simple Mail Transfer Protocol) se utiliza para enviar correos electrónicos y utiliza dos puertos específicos:

Puerto 25: Es el puerto estándar para la comunicación SMTP no cifrada. Sin embargo, en la actualidad, este puerto a menudo se utiliza solo para la comunicación entre servidores de correo y no para clientes de correo.

Puerto 587: Es el puerto estándar para SMTP con cifrado TLS/SSL, conocido como "SMTPS" o "SMTP sobre SSL". Este puerto se utiliza principalmente para la comunicación segura entre clientes de correo y servidores de correo. Por lo tanto, es comúnmente utilizado para el envío de correos electrónicos a través de clientes de correo, como Outlook, Thunderbird, etc.

    En Resumen:

  • Puerto 25 (no seguro)
  • Puerto 857 (seguro)

Protocolo POP3

El protocolo POP3 (Post Office Protocol version 3) se utiliza para recibir correos electrónicos de un servidor de correo. Aquí te muestro los puertos típicamente utilizados para POP3:

Puerto 110: Este es el puerto estándar para la comunicación POP3 no cifrada. Se utiliza para la comunicación no segura entre el cliente de correo y el servidor de correo.

Puerto 995: Este es el puerto estándar para POP3 con cifrado TLS/SSL, conocido como "POP3S" o "POP3 sobre SSL". Se utiliza para la comunicación segura entre el cliente de correo y el servidor de correo utilizando cifrado.

    En Resumen:

  • Puerto 110 (no seguro)
  • Puerto 995 (seguro)

Protocolo IMAP

El protocolo IMAP (Internet Message Access Protocol) se utiliza para acceder a correos electrónicos almacenados en un servidor de correo. Aquí te indico los puertos típicamente utilizados para IMAP:

Puerto 143: Este es el puerto estándar para la comunicación IMAP no cifrada. Se utiliza para la comunicación no segura entre el cliente de correo y el servidor de correo.

Puerto 993: Este es el puerto estándar para IMAP con cifrado TLS/SSL, conocido como "IMAPS" o "IMAP sobre SSL". Se utiliza para la comunicación segura entre el cliente de correo y el servidor de correo utilizando cifrado.

    En Resumen:

  • Puerto 143 (no seguro)
  • Puerto 993 (seguro)

Protocolo MIME

El protocolo MIME (Multipurpose Internet Mail Extensions) no utiliza puertos específicos. MIME es un estándar que extiende el formato de correo electrónico permitiendo la inclusión de elementos multimedia, texto enriquecido y otras características en los mensajes de correo electrónico.

MIME se basa en otros protocolos, como SMTP (Simple Mail Transfer Protocol) para enviar correos electrónicos y POP3/IMAP para recibir correos electrónicos, pero no define puertos específicos. En lugar de eso, MIME se integra en estos protocolos para permitir el envío y la recepción de correos electrónicos que contienen texto enriquecido, imágenes, archivos adjuntos y otros tipos de contenido multimedia.

    En Resumen:

  • No hay puertos exclusivos para el protocolo MIME, ya que es una extensión de otros protocolos de correo electrónico, y estos utilizan los puertos que ya se han mencionado para SMTP (25, 587), POP3 (110, 995), e IMAP (143, 993) según la necesidad de seguridad y cifrado.

Protocolo FTP

El protocolo FTP (File Transfer Protocol) utiliza dos puertos distintos para su funcionamiento:

Puerto 21 (FTP Control Port): Este es el puerto de control, también conocido como el puerto de comandos. Se utiliza para establecer la conexión inicial entre el cliente y el servidor FTP. Todas las instrucciones de control, como iniciar sesión, listar directorios y solicitar transferencias de archivos, se realizan a través de este puerto.

Puertos de datos (varios): Los puertos de datos se utilizan para la transferencia real de archivos. FTP utiliza diferentes puertos de datos dependiendo de la configuración y el modo de transferencia. Para conexiones FTP activas, el servidor utiliza el puerto 20 para enviar datos al cliente. Para conexiones FTP pasivas, el servidor elige un puerto de datos no reservado y lo comunica al cliente.

    En Resumen:

  • Puerto 21 (seguro)
  • Puertos de datos (medianamente seguro)

Protocolo FTPS

FTPS (FTP Seguro) es una extensión del protocolo FTP que utiliza cifrado SSL/TLS para proporcionar seguridad en las transferencias de archivos. FTPS puede utilizar dos métodos para cifrar la conexión: FTPS implícito y FTPS explícito.

FTPS Implícito: Este método utiliza el puerto 990 para la conexión de control y el puerto 989 para la conexión de datos. La conexión implícita significa que se espera que la conexión esté cifrada desde el principio.

FTPS Explícito: En este método, la conexión de control comienza en el puerto FTP estándar, que es el puerto 21. Luego, se negocia la seguridad mediante comandos específicos (por ejemplo, AUTH, PROT) y se inicia una conexión segura mediante SSL/TLS para la transferencia de datos.

    En Resumen:

FTPS Implícito

  • Puerto 990 para la conexión de control (FTP Control Port).
  • Puerto 989 para la conexión de datos (FTP Data Port).

FTPS Explícito

  • Puerto 21 para la conexión de control (FTP Control Port). La conexión segura se inicia más tarde.
  • Puerto 20 para la conexión de datos (FTP Data Port).

Protocolo SFTP

SFTP (SSH File Transfer Protocol) es un protocolo seguro y cifrado para transferir archivos sobre SSH (Secure Shell). A diferencia de FTPS, que es una extensión de FTP que utiliza SSL/TLS, SFTP es un protocolo completamente distinto que proporciona seguridad utilizando SSH.

SFTP utiliza por defecto el puerto 22 para establecer la conexión y transferir archivos. Es importante tener en cuenta que este puerto puede variar si se ha modificado la configuración en el servidor para utilizar un puerto diferente.

    En Resumen:

  • Puerto 22 por defecto (aunque puede configurarse para usar otros puertos si es necesario).

Protocolo BitTorrent

BitTorrent es un protocolo de transferencia de archivos peer-to-peer (P2P) que utiliza múltiples puertos para diferentes funciones dentro del protocolo. A continuación se detallan los puertos comúnmente asociados con BitTorrent

    En Resumen:

Puertos TCP:

  • 6881-6889: Estos son los puertos comúnmente asociados con la comunicación inicial entre el cliente y los peers. Tradicionalmente, el puerto 6881 era ampliamente utilizado, pero con el tiempo se extendió hasta el puerto 6889 para acomodar más conexiones.

Puertos UDP:

  • 6881-6889: Al igual que con TCP, los puertos UDP en el rango 6881-6889 se utilizan para la comunicación entre peers, especialmente para el seguimiento de torrents y la comunicación DHT (Distributed Hash Table).

Protocolo eDonkey

El protocolo eDonkey (usado en la red eDonkey2000) es un protocolo de red de intercambio de archivos peer-to-peer (P2P) que permite la transferencia de archivos entre usuarios. Este protocolo utiliza varios puertos para diferentes funciones en la red. Aquí te proporciono información sobre algunos de los puertos comúnmente asociados con el protocolo eDonkey

    En Resumen:

Puertos TCP:

  • TCP 4662: Este es el puerto principal utilizado por el protocolo eDonkey para la comunicación entre clientes (servidor y peer) en la red.

Puertos UDP:

  • UDP 4665: Este es el puerto utilizado para buscar archivos y obtener información de servidores en la red eDonkey.
  • UDP 4672: Este puerto es utilizado para la comunicación de servidor a servidor (se utiliza para conectar servidores en la red eDonkey).

Protocolo FastTrack

El protocolo FastTrack es un protocolo de red de intercambio de archivos peer-to-peer (P2P) utilizado en aplicaciones de intercambio de archivos como Kazaa y Grokster. Aunque se usó ampliamente en el pasado, su uso ha disminuido en los últimos años.

Debido a esta naturaleza dinámica y variada de los puertos utilizados por FastTrack, no hay un rango de puertos específico ampliamente reconocido que se pueda asociar de manera definitiva con este protocolo. Si estás interesado en investigar más sobre los puertos utilizados por FastTrack en un momento específico, te recomiendo utilizar herramientas de análisis de tráfico de red o consultar la documentación técnica específica del software que estés utilizando.

    En Resumen:

  • El protocolo FastTrack no utiliza puertos fijos y predefinidos como otros protocolos P2P, como BitTorrent o eDonkey. En lugar de eso, utiliza técnicas de "puerto dinámico" para el intercambio de archivos, lo que significa que los puertos que se utilizan pueden variar dinámicamente.

Protocolo IRC

El protocolo IRC (Internet Relay Chat) es un protocolo de comunicación en tiempo real basado en texto que se utiliza para la comunicación en grupos a través de canales en Internet. IRC utiliza varios puertos para diferentes funciones dentro del protocolo. Aquí están los puertos más comunes utilizados por IRC:

Es importante mencionar que si estás configurando o administrando un servidor IRC o un cliente IRC, puedes elegir utilizar diferentes puertos según tus necesidades de seguridad y configuración. Pero, en términos generales, el puerto 6667 para IRC no cifrado y el puerto 6697 para IRC cifrado son los más comúnmente utilizados.

    En Resumen:

  • Puerto TCP 6667: Este es el puerto predeterminado para conexiones sin cifrado. Es el puerto estándar para la comunicación no segura entre clientes y servidores IRC.
  • Puerto TCP 6697: Este es el puerto utilizado para conexiones seguras mediante SSL/TLS. Es el puerto estándar para la comunicación segura entre clientes y servidores IRC utilizando cifrado.

Protocolo XMMP

El protocolo XMPP (Extensible Messaging and Presence Protocol), utilizado para mensajería instantánea y presencia, utiliza puertos específicos para la comunicación entre clientes y servidores. Aquí te muestro los puertos estándar más comúnmente asociados con XMPP

Estos puertos son los más comúnmente utilizados para XMPP, pero es importante mencionar que XMPP es altamente configurable y es posible cambiar estos puertos en la configuración del servidor XMPP según las necesidades y políticas de la implementación.

Si estás configurando un servidor o cliente XMPP, asegúrate de verificar y configurar los puertos apropiados de acuerdo con tus requisitos de seguridad y configuración.

    En Resumen:

  • Puerto TCP 5222: Este es el puerto estándar para conexiones client-to-server sin cifrado. Es utilizado para la comunicación entre un cliente XMPP y un servidor XMPP.
  • Puerto TCP 5223: Este es el puerto estándar para conexiones client-to-server cifradas mediante SSL/TLS (XMPP sobre SSL). Es utilizado para la comunicación segura entre un cliente XMPP y un servidor XMPP utilizando cifrado SSL/TLS.
  • Puerto TCP 5269: Este es el puerto utilizado para la comunicación entre servidores (server-to-server). Permite que los servidores XMPP se comuniquen entre sí para enrutar mensajes y presencia.

Protocolo MTProto

MTProto es un protocolo desarrollado por Telegram Messenger para la comunicación segura entre clientes y servidores en su plataforma de mensajería. Telegram utiliza dos versiones principales de MTProto: MTProto 1.0 y MTProto 2.0.

Estos puertos son los más comúnmente utilizados para XMPP, pero es importante mencionar que XMPP es altamente configurable y es posible cambiar estos puertos en la configuración del servidor XMPP según las necesidades y políticas de la implementación.

Si estás configurando un servidor o cliente XMPP, asegúrate de verificar y configurar los puertos apropiados de acuerdo con tus requisitos de seguridad y configuración.

    En Resumen:

MTProto 1.0:

  • Puerto TCP 443: Este es el puerto estándar utilizado para MTProto 1.0. La elección de este puerto permite que el tráfico MTProto se asemeje al tráfico HTTPS, lo que ayuda a sortear bloqueos o restricciones impuestas en algunos entornos.

MTProto 2.0:

  • Puerto TCP 80: Telegram ha implementado MTProto 2.0 para que funcione sobre el puerto 80, que es comúnmente utilizado para tráfico HTTP no cifrado. Esta configuración también ayuda a evadir bloqueos o restricciones al hacer que el tráfico de MTProto 2.0 parezca tráfico HTTP convencional.
